PROPERTY DETAILS — ONE-FILE CONTROL PANEL

Copy property-details-template.json into the property folder and rename it:

property-details.json

Edit that one file whenever the public project information changes.

PRICING LOGIC
- "price": null
  The website displays Contact for pricing and opens the Contact Us popup.

- "price": 125000
  The website displays $125,000.

- A price plus a non-empty "checkout_url"
  The project action becomes Buy Now and opens the checkout link.

- A price without a checkout URL
  The numeric price displays, but Request Details remains the action so the website never sends a customer to a broken payment link.

- "show_contact_with_buy_button": true
  Shows both Buy Now and Request Details when a checkout URL is configured.

CATEGORY VALUES
- concept-home
- transformation
- property

COMMON STATUS EXAMPLES
- available
- coming-soon
- under-contract
- sold
- archived

IMPORTANT JSON RULES
- Keep double quotes around text.
- Keep commas between entries.
- Use true or false without quotes.
- Use null without quotes when no public price is available.
- Do not add comments inside JSON.

The property folder name controls the permanent URL. Changing "title" changes the displayed project name everywhere without changing the URL.
